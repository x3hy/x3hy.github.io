Project version control with Git+Make

Posted 2026-06-29T22:20:08.842Z ago

I have a lot of projects. Recently I've found a great method for version control that is almost fully automatic. I found that you can display the GIT version by running git describe --tags --always. This command outputs the git commit hash or, if a new version has just been released and no new commits have occured - it shows the version number (as set in the releases page in github). I found that I could easily incorperate this command into the displayed version control of my projects by writing the version to a file and then inserting that value into the README. I decided to modify this into a Makefile recipe that runs at build time:

.version: README trcs git describe --tags --always 2>/dev/null >$@ sed -i '1s/.*/PROGRAM_NAME-$(shell cat .version/' $(firstword $^)

That code will generate a .version and the contents of the first line of the README file will be replaced by "PROGRAM_NAME-version".

I do think that this is a very useful system as it resolves the need for you to manually update the version values (until you create a new release on GitHub).
